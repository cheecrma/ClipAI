# ClipAI — 클립보드 단축키 AI 어시스턴트

> Windows 상주형 트레이 앱. 마우스로 드래그한 텍스트를 단축키 한 번으로
> **요약 / 번역 / 리스트 정리** 하고, 깔끔한 다크 팝업에 결과를 스트리밍으로 띄운다.
> 최종 산출물은 단독 실행 `.exe`. 목표 응답: 짧은 텍스트 기준 **10초 이내**.

> ⚠️ **이 문서는 초기 "설계 명세"입니다 (설계 의도/근거 기록용).**
> 실제 구현·현재 상태·사용법은 **[README.md](README.md)** 를 보세요. 초기 명세와 달라진 주요 점:
> - **단축키**: (명세) A=요약/T=번역/S=리스트 → (현재) **A=요약 / S=리스트 / D=번역** + F=반박 / X=보고서
> - **기본 모델**: (명세) EXAONE 3.5 2.4B → (현재) **Gemma 2 2B**(상업 사용 가능). EXAONE는 개인/비상업용 대안.
> - **백엔드**: 배포 빌드는 명세대로 **llama-cpp-python**(CPU/AVX2, 0.3.2 고정)으로 완성.
>   초기 프로토타입의 **Ollama 백엔드 코드는 제거**(독립 실행 확정) → `requests` 의존도 제거.
> - **모드/프롬프트**: 이제 `config.toml` 의 `[[modes]]` 로 **코드 수정 없이 커스터마이즈** 가능 (명세엔 `[hotkeys]` 였음).
> - **팝업**: 사각 모서리 + 카톡형(좁고 긴) + 4변 드래그 리사이즈. 포커스 아웃 자동 닫힘은 비활성.

## 1. 목표와 범위

### 1.1 핵심 시나리오
사용자가 네이버 기사·문서·코드 등 임의의 앱에서 텍스트를 **드래그로 선택한 상태**에서
전역 단축키를 누르면, 선택 영역이 자동 캡처되어 모드에 맞는 AI 처리가 실행되고
결과가 팝업으로 즉시 표시된다.

| 단축키 | 모드 | 동작 |
|---|---|---|
| `Ctrl + Alt + A` | summarize | 선택 텍스트 요약 |
| `Ctrl + Alt + T` | translate | 한국어↔영어 자동 감지 번역 (한국어면 영어로, 영어면 한국어로) |
| `Ctrl + Alt + S` | listify | 핵심을 불릿 리스트로 구조화 정리 |

### 1.2 비기능 요구사항
- **체감 무지연**: 단축키 → 첫 토큰 노출까지 1~2초, 전체 완료 10초 이내.
- **깔끔한 UI**: 다크 테마, 모서리 둥근 미니 팝업, 마우스 커서 근처에 출현.
- **단독 배포**: `.exe` 더블클릭으로 실행. 별도 서버(Ollama) 구동 불필요.
- **로컬 추론**: 사내 보안상 텍스트가 외부로 나가지 않음(전부 온디바이스).

### 1.3 비범위 (이번 버전 제외)
- 클라우드 LLM 호출, 멀티모달(이미지), 대화형 후속 질문(원샷 처리만), 설정 GUI.
  → 설정은 `config.toml`로만. 후속 확장 항목은 §12 참고.

## 2. 아키텍처 결정 (가장 중요)

> "Ollama API보다 최적화"의 핵심은 HTTP 제거가 아니다.
> localhost HTTP 왕복은 수 ms라 체감이 없다. **진짜 병목은 아래 3가지다.**

### 2.1 레이턴시 병목과 대응
| 병목 | 잘못된 구조 | 본 설계 |
|---|---|---|
| 모델 콜드 스타트 (1~10초) | 호출 때마다 모델 로드 | **상주 프로세스가 시작 시 1회 로드 후 메모리 유지** |
| 프로세스 기동 비용 | 단축키마다 새 python 실행 | 트레이 앱 1개가 계속 떠서 추론만 트리거 |
| 팝업 생성 지연 | 매번 윈도우 new | **숨김 윈도우 미리 생성 → show만** |
| 체감 대기 | 완료 후 한 번에 출력 | **토큰 스트리밍**으로 첫 글자 즉시 |

### 2.2 추론 백엔드: llama-cpp-python (인프로세스) 채택
- **Ollama 대비 장점**: 별도 서비스 데몬 의존성 제거 → `.exe` 단독 배포에 유리,
  프롬프트 캐시/샘플링 직접 제어, 모델 로드 타이밍 통제 가능.
- **Ollama를 써도 되는 경우**: 개발 초기 빠른 프로토타입.
  이때는 반드시 `keep_alive: -1`(모델 상주) + `stream: true`로 호출.
  → 즉, Ollama로 만들든 llama.cpp로 만들든 **"모델 상주 + 스트리밍"이 본질**.
- **결론**: 프로토타입은 Ollama 허용, **배포 빌드는 llama-cpp-python으로 통일**.

### 2.3 단일 상주 프로세스 구조
```
[트레이 아이콘 (pystray)]
        │
        ├── 전역 단축키 리스너 (keyboard / RegisterHotKey)
        │
        ├── LLM 엔진 (llama-cpp-python, 모델 1회 로드 후 상주)
        │
        └── 팝업 윈도우 (미리 생성, 숨김 → show/hide 재사용)
```
- 단축키 핸들러는 **즉시 반환**하고, 추론은 **워커 스레드**에서 실행.
- 추론 토큰은 큐를 통해 **UI 스레드로 마샬링**해 팝업에 흘려보냄(스레드 안전).

## 3. AI 모델 선택

### 3.1 권장 (한국어 특화 · 온디바이스)
| 우선순위 | 모델 | 크기 | 비고 |
|---|---|---|---|
| **1순위** | `EXAONE-3.5-2.4B-Instruct` (Q4_K_M GGUF) | 2.4B | 한·영 이중언어, 공식 GGUF, llama.cpp/Ollama 셋업 간단, 32K 컨텍스트 |
| 2순위 | `EXAONE-4.0-1.2B` (비추론 모드, GGUF) | 1.2B | 가장 빠름/가벼움. 단 llama.cpp 최신 빌드 필요, **반드시 reasoning OFF** |
| 대안 | `Qwen2.5-3B-Instruct` (Q4_K_M) | 3B | 구조화 출력(리스트 모드)에 강함, 다국어 양호 |

### 3.2 선택 근거
- 요약·번역·리스트는 "즉답" 작업이라 **추론(reasoning) 모드는 레이턴시만 키운다.**
  EXAONE 4.0을 쓸 경우 **비추론 모드 고정** 필수.
- EXAONE 4.0 1.2B 한국어 대화 시 `temperature=0.1` 권장(코드 스위칭 방지).
- CPU 전용 환경이면 **1.2B가 10초 목표 달성에 가장 안전**.
  NVIDIA GPU가 있으면 CUDA 빌드로 2.4B도 여유.

### 3.3 하드웨어별 가이드
- **GPU 있음(사내 PC에 NVIDIA)**: llama-cpp-python CUDA 빌드, `n_gpu_layers=-1`, EXAONE 3.5 2.4B.
- **CPU 전용**: EXAONE 4.0 1.2B 또는 2.4B + 출력 길이 제한, `n_threads`를 물리코어 수로.
- **(선택) 집 DGX Spark 연동**: 사무실에서 네트워크로 닿지 않으면 비활성.
  닿는 환경이면 `config.toml`에서 `backend = "remote"`로 OpenAI 호환 엔드포인트 지정 가능하게 설계(확장 포인트).

## 4. 기술 스택

| 영역 | 선택 | 이유 |
|---|---|---|
| 언어 | Python 3.11 | PyInstaller 안정성, 라이브러리 호환 |
| 추론 | `llama-cpp-python` | 인프로세스, 스트리밍, 단독 배포 |
| 전역 단축키 | `keyboard` (1순위) / win32 `RegisterHotKey` (견고) | 간단함 vs 권한·충돌 회피 |
| 클립보드 | `pyperclip` 또는 `win32clipboard` | 선택 텍스트 캡처/복원 |
| 키 입력 시뮬 | `keyboard` 또는 `pynput` | Ctrl+C 전송 |
| UI | `customtkinter` (1순위) / `PySide6` (고급 폴리시) | 다크테마 내장·경량 vs 미려함·exe 큼 |
| 트레이 | `pystray` + `Pillow` | 트레이 아이콘/메뉴 |
| 패키징 | `PyInstaller` (`--onefile` 또는 onedir) | exe 생성 |
| 설정 | `tomllib`(표준) | config.toml |

> UI 라이브러리 메모: 시작 비용은 **상주 앱이라 1회만** 지불 → 무거운 Qt도 허용 가능.
> 다만 exe 크기·번들 난이도는 customtkinter가 유리. **1순위 customtkinter로 진행, 폴리시 부족하면 PySide6 검토.**

## 5. 동작 흐름 (단축키 → 결과)

```
1. 사용자가 텍스트 드래그 선택
2. Ctrl+Alt+A 누름
3. 핸들러:
   a. (선택) 현재 클립보드 백업
   b. Ctrl+C 시뮬레이션 전송
   c. 80~150ms 대기 (앱이 클립보드 채울 시간)
   d. 클립보드에서 선택 텍스트 읽기
   e. (선택) 원래 클립보드 복원
4. 텍스트 비었으면 → "선택된 텍스트 없음" 토스트 후 종료
5. 팝업 즉시 show (커서 근처), 상단에 모드명 + 경과시간 타이머 표시
6. 워커 스레드에서 LLM 스트리밍 추론 시작
7. 토큰을 큐→UI 스레드로 전달, 팝업 텍스트박스에 실시간 append
8. 완료 시 타이머 정지, 하단에 [복사] [닫기] 버튼 활성
9. Esc 또는 포커스 아웃 시 hide (윈도우는 파괴하지 않고 재사용)
```

### 5.1 선택 캡처 주의점 (중요)
- Ctrl+C 후 **즉시 읽으면 빈 값**이 나오는 앱이 많다 → 짧은 대기 필수(폴링 권장:
  최대 ~300ms 동안 클립보드 변화 폴링).
- 일부 앱(보안 PDF 등)은 복사 불가 → 빈 결과 정중히 안내.
- 클립보드 백업/복원은 사용자의 기존 복사 내용을 보호(선택 구현, 기본 ON 권장).

## 6. 모드별 프롬프트 설계

`prompts.py`에 모드별 시스템 프롬프트 분리. 모두 **간결·저온도**로 레이턴시/일관성 확보.

### 6.1 summarize (`Ctrl+Alt+A`)
- system: "너는 한국어 요약 도우미다. 입력 텍스트의 핵심만 3~5문장으로,
  군더더기·인사말 없이 요약하라. 원문 언어로 답하라."
- params: `temperature=0.2`, `max_tokens≈256`

### 6.2 translate (`Ctrl+Alt+T`)
- 언어 자동 감지: 입력의 한글 비율로 판단(한글 우세 → 영어로, 그 외 → 한국어로).
- system(KO→EN): "다음 한국어를 자연스러운 영어로 번역하라. 설명 없이 번역문만 출력."
- system(EN→KO): "다음 영어를 자연스러운 한국어로 번역하라. 설명 없이 번역문만 출력."
- params: `temperature=0.3`, `max_tokens≈ 입력 길이 비례`

### 6.3 listify (`Ctrl+Alt+S`)
- system: "입력 내용을 핵심 항목 불릿 리스트로 정리하라. 각 항목은 한 줄,
  중복 제거, 논리 순서. 마크다운 '- ' 형식. 불필요한 머리말 금지."
- params: `temperature=0.2`, `max_tokens≈300`
- Qwen2.5-3B가 이 모드에서 특히 안정적.

> 공통: 출력에 "요약:" 같은 머리말이 붙지 않도록 프롬프트로 강하게 억제.
> stop 토큰/길이 제한으로 장황함 방지.

## 7. UI 설계

- **형태**: 폭 ~420px, 높이 가변(내용 따라, 최대치 후 스크롤). 둥근 모서리, 다크.
- **출현 위치**: 마우스 커서 근처(화면 경계 보정). always-on-top.
- **구성**:
  - 헤더: 모드 아이콘+이름(예: "�summary 요약") · 우측에 경과시간 `0.0s ▸` 실시간 갱신
  - 본문: 스트리밍 텍스트(읽기 좋은 폰트, 줄간격)
  - 푸터: `[복사]` `[닫기]` (단축키 안내 작게)
- **상태 표시**: 추론 중 상단 얇은 인디케이터/타이머 → 완료 시 정지.
- **재사용**: 윈도우 1개를 hide/show. 모드 전환 시 내용만 교체.
- **키**: `Esc` 닫기, `Ctrl+C`(팝업 포커스 시) 결과 복사.

> 기존 `clip_assistant.py`의 다크 팝업 + 경과시간 인디케이터 자산을 재활용/이식.

## 9. 프로젝트 구조 (제안)
```
clipai/
├── CLAUDE.md                # 본 문서
├── config.toml              # 모델 경로, 단축키, 백엔드 등
├── requirements.txt
├── src/
│   ├── main.py              # 진입점: 트레이 + 단축키 + 엔진 + UI 와이어링
│   ├── engine.py            # LLM 로드/스트리밍 추론 (llama-cpp-python)
│   ├── hotkeys.py           # 전역 단축키 등록/해제
│   ├── clipboard.py         # 선택 캡처(Ctrl+C 시뮬+폴링), 백업/복원
│   ├── prompts.py           # 모드별 시스템 프롬프트 + 언어감지
│   ├── popup.py             # 다크 팝업 윈도우(재사용, 스트리밍 렌더)
│   ├── tray.py              # pystray 아이콘/메뉴(종료, 재시작, 모델표시)
│   └── workers.py           # 스레드/큐(추론→UI 마샬링)
├── models/                  # GGUF 파일 (exe와 별도 외부 배치 권장)
└── build/
    └── clipai.spec          # PyInstaller 스펙
```

### config.toml 예시
```toml
[model]
backend   = "llamacpp"        # "llamacpp" | "ollama" | "remote"
path      = "models/EXAONE-3.5-2.4B-Instruct-Q4_K_M.gguf"
n_ctx     = 4096
n_gpu_layers = -1             # GPU 없으면 0
n_threads = 8                 # 물리코어 수

[hotkeys]
summarize = "ctrl+alt+a"
translate = "ctrl+alt+t"
listify   = "ctrl+alt+s"

[ui]
theme = "dark"
width = 420
restore_clipboard = true
```

## 10. .exe 패키징 (PyInstaller)
- **모델 GGUF는 exe에 번들하지 말 것**(1GB+ → 거대 exe). `models/` 외부 폴더 로드.
  실행 경로 기준 상대경로 + `sys._MEIPASS` 인지 로직.
- llama-cpp-python의 `llama.dll`/`ggml` 바이너리는 `--add-binary` 또는 hook으로 포함 확인.
- `customtkinter`/`pystray`/`Pillow` 데이터 파일 누락 주의(hidden-import/datas).
- **onedir 우선 권장**(onefile은 매 실행 임시 추출로 느려 상주앱에 불리). 배포는 폴더 zip.
- 트레이 아이콘용 `.ico` 포함.
- 빌드 후 **클린 PC에서 DLL 누락 테스트** 필수.

## 11. 구현 마일스톤
- **M0 — 스파이크**: Ollama(`keep_alive:-1`,`stream`)로 단축키→캡처→콘솔 출력까지 동작 확인.
- **M1 — 코어**: 상주 트레이 앱 + 3개 단축키 + 선택 캡처(폴링/복원).
- **M2 — 엔진 전환**: llama-cpp-python 인프로세스 로딩 + 스트리밍.
- **M3 — UI**: 다크 팝업 재사용, 토큰 스트리밍 렌더, 경과시간, 복사/닫기.
- **M4 — 모드 완성**: summarize/translate(언어감지)/listify 프롬프트 튜닝.
- **M5 — 최적화**: §8 체크리스트로 10초 목표 검증, 모델 1.2B/2.4B 비교 벤치.
- **M6 — 패키징**: PyInstaller onedir 빌드, 클린 PC 검증, 배포 zip.

## 12. 리스크 / 주의
- `keyboard` 라이브러리는 환경에 따라 **관리자 권한**이 필요하거나 다른 단축키와 충돌
  가능 → 안 되면 win32 `RegisterHotKey`로 전환.
- 선택 캡처가 일부 앱(보안 문서)에서 막힘 → 빈 결과 안내 처리.
- EXAONE 4.0은 llama.cpp 최신 빌드/chat template 이슈 가능 → 도입 시 버전 확인.
  안정성 우선이면 EXAONE 3.5 2.4B로 시작.
- CPU 환경에서 2.4B가 10초를 넘기면 1.2B로 하향 또는 출력 길이 축소.
- 사내 보안 정책: 모델/텍스트가 외부 전송되지 않음을 보장(로컬 추론 고정).
